# Database Transaction Management

If all operations succeed, `COMMIT` permanently saves the changes.

## Additional Notes

- **@@ERROR**: A system variable in SQL Server used to check if the last statement executed successfully. Other databases like MySQL or PostgreSQL use different mechanisms (`ERROR` or `EXCEPTION`) for error handling.
- **Logical Errors**: Besides system errors, logical errors (e.g., insufficient balance, out-of-stock issues) should be handled using `ROLLBACK` to ensure business logic correctness.

## 3.4. Summary

`ROLLBACK` is a key function in database management systems that ensures data integrity, consistency, and atomicity by undoing all changes made during a transaction. It enhances system reliability and is widely used in scenarios such as bank transfers, order systems, data imports, and system failure recovery.
