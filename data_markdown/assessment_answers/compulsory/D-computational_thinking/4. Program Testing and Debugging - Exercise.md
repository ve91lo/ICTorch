# Comparison of Sorting Algorithms: Bubble Sort vs. Merge Sort

## 1.8 Memory Usage Comparison

### Answer:
- **Bubble Sort** uses less memory compared to **Merge Sort**.
- Bubble Sort sorts the list in place, meaning it does not require additional memory beyond the list itself.
- **Merge Sort**, however, requires extra memory to create temporary lists during the merging process.
- This additional memory usage is proportional to the size of the input list, making Merge Sort use more memory compared to Bubble Sort.

## 1.9 Implementation Complexity Comparison

### Answer:
- **Bubble Sort** is generally easier to implement compared to **Merge Sort**.
- Bubble Sort involves straightforward nested loops that compare and swap adjacent elements, making the code simple and easy to understand.
- **Merge Sort** requires:
- Dividing the list into smaller parts
- Recursively sorting each part
- Merging them back together
- This involves more complex logic and handling of multiple lists, making Merge Sort more challenging to implement than Bubble Sort.
