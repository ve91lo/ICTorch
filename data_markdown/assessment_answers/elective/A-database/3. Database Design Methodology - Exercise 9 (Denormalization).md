# 1.2. Describe the Trade-offs and Considerations of Denormalization in a Database System

## Answer:

While denormalization can provide performance benefits, it comes with several trade-offs:

- **Increased Redundancy**: Denormalized databases may contain duplicate data, which increases storage requirements.
- **Update Anomalies**: Because the same data may appear in multiple places, maintaining consistency can be more challenging. If one instance of the data is updated but others are not, it can lead to data inconsistencies.
- **More Complex Writes**: Updates, inserts, and deletes can become more complex, as changes need to be propagated across multiple copies of the same data.
- **Data Integrity Risks**: Denormalization can introduce potential risks to data integrity, as the system may not enforce the same level of consistency as a fully normalized design.
