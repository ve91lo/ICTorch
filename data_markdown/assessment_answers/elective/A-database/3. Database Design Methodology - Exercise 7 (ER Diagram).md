# Database Relationships

## Relationships

### Book to Borrow (is_borrowed_by)
- This relationship indicates that a book can be borrowed by one or more members.
- A book may appear in many borrow records, as it can be borrowed multiple times.

### Member to Borrow (borrows)
- This represents a one-to-many relationship between MEMBER and BORROW.
- The `1--o{` symbol indicates that a single member can borrow multiple books.
- The relationship is labeled "borrows", meaning that each member may borrow many books over time.
- A member may have many entries in the BORROW table, each representing a separate borrowing event for different books.

## Entity Relationship Diagram

#### BORROW
| Field        | Type   | Key  |
|--------------|--------|------|
| BookID      | int    | PK   |
| MemberID    | int    | PK   |
| BorrowDate   | date   |      |
| ReturnDate   | date   |      |

## Summary
- A **BOOK** can be borrowed by many **MEMBERS** over time, hence the BOOK to BORROW relationship is one-to-many.
- A **MEMBER** can borrow many **BOOKS**, hence the MEMBER to BORROW relationship is also one-to-many.
- The BORROW table is the junction table connecting BOOK and MEMBER. Each record in the BORROW table represents a borrowing transaction, which includes details like the borrow date, return date, and the associated book and member.
