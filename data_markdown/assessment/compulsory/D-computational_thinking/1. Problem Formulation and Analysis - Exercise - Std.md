# ICT Educational Material

## 1. Define the Problem and Scope for a System Designed to Manage a Library's Book Catalog
- **Core Needs**:
- Efficient cataloging of books
- User-friendly interface for searching and borrowing
- Management of book loans and returns
- **Challenges**:
- Keeping track of book availability
- Managing user accounts and permissions
- Integrating with existing library systems
- **Objectives**:
- Streamline the book lending process
- Provide accurate and up-to-date information on book inventory
- Enhance user experience for library patrons

## 2. Identify the Required Inputs and Outputs and Describe the Method to Calculate the Loan Interest
- **Inputs**:
- Principal amount (loan amount)
- Interest rate
- Loan duration
- **Outputs**:
- Total interest payable
- Total amount to be repaid
- **Method to Calculate Loan Interest**:
- Formula:
\[
\text{Total Interest} = \text{Principal} \times \text{Interest Rate} \times \text{Time}
\]
- **Example of Required Installments**:
- Monthly payments can be calculated using the formula for an amortizing loan.

## 3. Explain how to Break a Complex Problem into Smaller, Manageable Sub-problems
- **Example**: Calculating a Student's Final Grade
- **Sub-problems**:
1. Determine the weight of each assignment
2. Calculate the average score for each assignment type
3. Combine the averages according to their weights
4. Round the final grade as per grading policy

## 4. Identify the Common Elements in Problems of Sorting Students by Height and by Weight
- **Common Elements**:
- Both require a list of students with associated measurements
- Sorting algorithms can be applied similarly
- Data structures (arrays/lists) can be used to store the information
- **Simplification**: Recognizing these elements allows for the reuse of sorting algorithms and data handling techniques.

## 5. Explain how Recognizing Similar Elements in Different Problems Can Help in Developing Solutions
- **Examples**:
- **Sorting Students by Height and Weight**: Both problems can utilize the same sorting algorithm, reducing the need to develop separate solutions.
- **Programming a Robot to Navigate Different Polygons**: Recognizing that navigation involves similar pathfinding algorithms can streamline the programming process.

## 6. For the Problem of Designing a Shopping Cart System for an Online Store
- **Outline of Steps**:
1. Identify user requirements (add, remove, view items)
2. Define the data structure for the cart
3. Break down functionalities (calculating totals, applying discounts)
4. Design the user interface for the cart
5. Implement backend logic for cart management

## 7. Given a Problem of Determining the Body Mass Index (BMI) to Maintain a Healthy Weight
- **Break it down into Sub-problems**:
1. Collect user data (height and weight)
2. Calculate BMI using the formula:
\[
\text{BMI} = \frac{\text{weight (kg)}}{\text{height (m)}^2}
\]
3. Interpret the BMI value (underweight, normal, overweight, obese)
4. Provide recommendations based on BMI category

## 8. Describe how to Set Clear Goals and Define the Scope of a Problem
- **Steps**:
1. Identify the purpose of the system (e.g., a simple calculator)
2. Define functionalities (addition, subtraction, multiplication, division)
3. Set limitations (no advanced functions)
- **Contribution to Solving the Problem**: Clear goals help focus development efforts and avoid scope creep.

## 9. What Are the Key Challenges and Limitations that Should Be Considered when Defining the Scope of a Problem?
- **Challenges**:
- Budget constraints: Limited resources may restrict features.
- Technological limitations: Existing infrastructure may not support new solutions.
- **Examples**:
- A project may need to cut features due to budget cuts.
- A system may not be able to integrate with outdated technology.
