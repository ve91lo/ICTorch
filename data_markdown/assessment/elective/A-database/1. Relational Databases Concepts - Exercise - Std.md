# Questions

## 2.1. What is an entity in a relational database, and how does it differ from an attribute? Provide an example of each in the context of a university database.

- **Entity**: An entity is a thing or object in the real world that is distinguishable from other objects. In a university database, an example of an entity could be a "Student".
- **Attribute**: An attribute is a property or characteristic of an entity. For the "Student" entity, an example of an attribute could be "Student ID".

## 2.2. Explain the concept of a primary key and a composite key. How are they used to ensure data integrity in a relational database?

- **Primary Key**: A primary key is a unique identifier for a record in a table. It ensures that each record can be uniquely identified.
- **Composite Key**: A composite key is a primary key that consists of two or more attributes. It is used when a single attribute is not sufficient to uniquely identify a record.
- **Data Integrity**: Both primary keys and composite keys help maintain data integrity by preventing duplicate records and ensuring that each record can be uniquely identified.

## 2.3. What is a foreign key, and how does it relate to referential integrity in a relational database? Provide an example.

- **Foreign Key**: A foreign key is an attribute in one table that refers to the primary key in another table. It establishes a relationship between the two tables.
- **Referential Integrity**: Referential integrity ensures that relationships between tables remain consistent. For example, if a "Course" table has a foreign key that references the "Student" table, it ensures that every course entry corresponds to a valid student.

## 2.4. Describe what is meant by domain integrity in a relational database. Why is it important? Provide an example of how domain integrity can be enforced.

- **Domain Integrity**: Domain integrity refers to the validity of data within a specific domain or range of values. It ensures that all entries in a column fall within a defined set of values.
- **Importance**: It is important because it maintains the accuracy and consistency of data.
- **Example of Enforcement**: Domain integrity can be enforced by using constraints, such as setting a range for a numeric field or restricting a text field to specific values (e.g., using an ENUM type).

## 2.5. How can indexes improve the performance of a relational database? Provide an example where an index might be necessary in a large database.

- **Indexes**: Indexes improve the performance of a database by allowing faster retrieval of records. They work like a book index, enabling quick access to data without scanning the entire table.
- **Example**: In a large database of "Employees", an index on the "Last Name" column would allow for quicker searches when querying employees by their last names.

## 2.6. Explain the purpose of primary keys and foreign keys in a relational database. Provide an example to illustrate their roles.

- **Primary Keys**: They uniquely identify each record in a table. For example, in a "Students" table, the "Student ID" can serve as a primary key.
- **Foreign Keys**: They create a link between two tables. For example, in a "Enrollments" table, a "Student ID" foreign key can reference the "Student ID" primary key in the "Students" table, establishing a relationship between the two.

## 2.7. Describe the difference between a one-to-many relationship and a many-to-many relationship in a relational database. Provide an example of each.

- **One-to-Many Relationship**: In this relationship, a single record in one table can be associated with multiple records in another table.
- **Example**: A "Professor" can teach multiple "Courses".

- **Many-to-Many Relationship**: In this relationship, multiple records in one table can be associated with multiple records in another table.
- **Example**: Students can enroll in multiple courses, and each course can have multiple students.

## 2.8. Outline the steps involved in creating a simple relational database for a library system. Identify the primary entities, their attributes, and how you would establish relationships between them.

1. **Identify Entities**:
- **Books**: Attributes - Book ID (Primary Key), Title, Author, ISBN.
- **Members**: Attributes - Member ID (Primary Key), Name, Membership Date.
- **Loans**: Attributes - Loan ID (Primary Key), Book ID (Foreign Key), Member ID (Foreign Key), Loan Date, Return Date.

2. **Establish Relationships**:
- A "Member" can have multiple "Loans" (One-to-Many).
- A "Book" can be associated with multiple "Loans" (One-to-Many).

## 2.9. What is the purpose of the ROLLBACK statement in SQL, and how does it affect a transaction?

- **ROLLBACK Statement**: The ROLLBACK statement is used to undo changes made during a transaction. It reverts the database to its previous state before the transaction began.
- **Effect on Transaction**: It ensures that if an error occurs or if the transaction needs to be canceled, all changes made during that transaction are discarded.

## 2.10. Describe a scenario in which you would use ROLLBACK in a database transaction. Include the steps that would be taken to handle the situation and the related SQL examples.

- **Scenario**: A bank transfer between two accounts.
1. Begin the transaction.
2. Deduct the amount from Account A.
3. If the deduction is successful, add the amount to Account B.
4. If any step fails (e.g., insufficient funds), execute ROLLBACK to revert changes.

```sql
BEGIN TRANSACTION;

UPDATE Accounts SET Balance = Balance - 100 WHERE AccountID = 'A';
IF @@ERROR &#x3C;> 0
BEGIN
ROLLBACK;
RETURN;
END

UPDATE Accounts SET Balance = Balance + 100 WHERE AccountID = 'B';
IF @@ERROR &#x3C;> 0
BEGIN
ROLLBACK;
RETURN;
END

COMMIT;
```

## 2.11. Explain how a ROLLBACK operation helps in maintaining database consistency and provide an example of its impact on a multi-step transaction.

- **Maintaining Consistency**: A ROLLBACK operation ensures that the database remains in a consistent state by undoing all changes made during a transaction if an error occurs.
- **Example**: In a multi-step transaction involving transferring funds, if the first step (deducting funds) is successful but the second step (adding funds) fails, ROLLBACK will revert the deduction, ensuring that no funds are lost or incorrectly allocated.
