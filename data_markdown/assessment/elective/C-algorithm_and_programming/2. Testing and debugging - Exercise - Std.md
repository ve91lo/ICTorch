# Questions

## 1.1. Explain what a rounding error is and provide a Python code snippet that demonstrates how rounding can lead to inaccuracies. What is the solution to avoid such errors?

A rounding error occurs when a number is approximated to a certain number of decimal places, leading to a loss of precision. This can result in inaccuracies in calculations.

### Python Code Snippet
```python
# Example of rounding error
value = 0.1 + 0.2
rounded_value = round(value, 1)
print(rounded_value)  # Output: 0.3
```

### Solution
To avoid rounding errors, it is recommended to use the `decimal` module in Python, which provides support for fast correctly-rounded decimal floating point arithmetic.

```python
from decimal import Decimal

value = Decimal('0.1') + Decimal('0.2')
print(value)  # Output: 0.3
```

## 1.4. What is a runtime error? Write a Python code snippet that demonstrates a runtime error.

A runtime error occurs while the program is running, often due to illegal operations, such as dividing by zero.

### Python Code Snippet
```python
# Example of runtime error
numerator = 10
denominator = 0
result = numerator / denominator  # This will raise a ZeroDivisionError
```

## 1.5. Explain the purpose of using stub procedures in testing. Provide a real-world example where stubs could be applied in a software development scenario.

Stub procedures are used in testing to simulate the behavior of complex modules that are not yet implemented or are difficult to test. They allow developers to test the interactions between components without needing the full implementation.

### Real-World Example
In a software development scenario, if a team is developing a payment processing system, they might use stubs to simulate the payment gateway. This allows them to test the integration of their application with the payment system without needing the actual payment gateway to be fully functional.

## 1.6. What are breakpoints, and how do they aid in the debugging process? Provide an example application.

Breakpoints are markers set in the code that allow the execution to pause at a specific line. This helps developers inspect the state of the application, including variable values and the flow of execution, making it easier to identify and fix bugs.

### Example Application
In an Integrated Development Environment (IDE) like PyCharm or Visual Studio Code, a developer can set a breakpoint on a line of code. When the program runs and hits that line, it will pause, allowing the developer to check the values of variables and step through the code line by line to understand the behavior of the program.
